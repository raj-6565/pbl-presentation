Problem Statement

Deepfake videos and manipulated digital media are increasingly used for misinformation, fraud, and identity impersonation. Most existing forensic tools rely on heavy deep learning models requiring powerful GPUs, making them impractical for general use. There is a need for a lightweight, interpretable, and deployable multimodal forensic system capable of analysing visual, audio, and metadata signals together.

Literature Review

FakeHunter, DeepWare Scanner, and Microsoft Video Authenticator achieve good accuracy but require heavy compute and lack explainability. Research shows multimodal analysis — combining visual, audio, and metadata signals — improves reliability over single-modality approaches.

Research Gap

No existing tool measures how reliable each input channel is before fusing evidence, or treats inter-module disagreement as an independent forensic signal. Our system addresses both gaps with a novel fusion strategy.

Visual Analysis

Extracts 30 evenly-spaced frames using OpenCV. Each frame passes through EfficientNet-B0 (pretrained on ImageNet) to produce a 1280-dimensional visual embedding. A Logistic Regression classifier trained on our dataset scores each frame 0 to 1. Video-level statistics are computed: mean, variance, ratio, and peak score.

EfficientNet-B0 Transfer learning
Audio Analysis

FFmpeg extracts the audio track. librosa computes MFCC features — low variance indicates synthetic speech — along with spectral centroid, energy variance, and silence ratio. If no audio track exists the module is automatically excluded with weight zero.

librosa FFmpeg
Metadata Analysis

ffprobe reads hidden technical properties: FPS, resolution, codec, bitrate, and frame rate mode. Re-encoded deepfakes leave characteristic traces — non-standard FPS, older codecs, variable frame rate — which are scored as anomalies.

ffprobe Encoding forensics
Fusion Model Random Forest · 300 trees

The three module outputs are summarised into 8 numerical features — visual mean, variance, ratio, peak, audio score, metadata score, FPS flag, and disagreement entropy. A Random Forest classifier trained on these 8 features produces the final authenticity probability. The system was evaluated on a proper 80/20 train-test split with no data leakage between splits.

Novel Contribution

QAEF — Quality-Aware Evidence Fusion — the key innovation of this project, addressing the gap in all existing multimodal deepfake detection tools.

01

Quality-aware weights

Before combining evidence, we measure how reliable each input channel is. Blurry video → visual weight is reduced. No audio → excluded automatically. Clean sharp video → visual weight increased. Weights adapt dynamically based on measured input quality — unlike existing tools which always use fixed 50/30/20 regardless.

02

Disagreement entropy

Shannon entropy measures how much the three modules conflict with each other. When visual says FAKE but metadata says REAL, that conflict is itself suspicious — a deepfake may fool one channel but not another. We treat this disagreement as an independent forensic signal, adding to the suspicion score when modules conflict strongly.

03

Explainability

Every decision comes with a complete forensic explanation: per-module anomaly scores, adaptive weight breakdown, entropy level, GradCAM++ heatmaps showing which frame regions triggered detection, and a natural language "Why this decision?" paragraph. No existing tool provides this level of interpretability.
